# 风扇控制

## 1. 硬件分配

| 信号 | 引脚 | 定时器配置 | 负载 |
| --- | --- | --- | --- |
| `FAN_MCU_PWM1` | PA8 | TIM1_CH1，2kHz PWM | 两个 4114N/2H4PU，四个电机水套散热 |
| `FAN_MCU_PWM2` | PB8 | TIM4_CH3，2kHz PWM | 一个 4114N/2H6P，逆变器散热器 |
| `FAN1_MCU_TACH` | PA0 | TIM2_CH1，下降沿输入捕获 | 4114N/2H4PU #1 |
| `FAN2_MCU_TACH` | PA6 | TIM3_CH1，下降沿输入捕获 | 4114N/2H4PU #2 |
| `FAN3_MCU_TACH` | PA7 | TIM3_CH2，下降沿输入捕获 | 4114N/2H6P |

TIM1 和 TIM4 的计数时钟均为 72MHz，`PSC=35`、`ARR=999`，PWM 频率为 2kHz。TIM2 和 TIM3 使用 1MHz 计数频率，输入捕获数字滤波值为 4。

PWM 控制经过 2N7002 反相。MCU 输出高电平时风扇控制线为低电平，风扇停止；MCU 输出低电平时风扇全速。软件接口和 CAN 报文统一使用风扇侧占空比：0%表示停止，100%表示全速。

三个 TACH 信号经过 TLP291 光耦，每个风扇每转输出两个脉冲：

```text
RPM = 30,000,000 / 相邻下降沿间隔(us)
```

## 2. 控制模式

固件有三种模式：

| 模式值 | 模式 | 行为 |
| ---: | --- | --- |
| 0 | 自动 | 根据 CANB 温度和当前策略计算两路目标占空比 |
| 1 | 手动 | 使用 CAN 命令给出的 PWM1、PWM2 目标占空比 |
| 2 | 关闭 | 两路目标均为0%，用于限时台架测试或远程停机 |

手动和关闭模式必须携带1至60秒的有效时间，并由上位机周期续发。超时后自动回到自动模式，防止通信中断后永久保持手动转速或永久关闭。温度曲线和失联策略只保存在 RAM 中，复位后恢复默认值。

## 3. 缓慢启动与占空比变化

所有启动路径均经过同一套缓慢变化处理，包括自动温控、温度失联处理、手动控制和强制全速策略：

- 默认上升速度为20%/s，从0%升到100%约需5秒；
- 默认下降速度为50%/s，从100%降到0%约需2秒；
- 两组风扇从0%开始运行时至少间隔2.5秒；如果一组刚开始上升，另一组先保持0%；
- PWM1和PWM2不会因为温度报文超时或远程命令而从0%直接跳到高占空比；
- 自动模式仍使用30%的最低稳定运行占空比，但启动时会从0%逐步经过较低占空比。

上升速度、下降速度和两路目标占空比可以通过 CAN 命令调整。2.5秒的两组启动间隔为固定保护值。

## 4. 自动温控

固件接收以下 CANB 标准帧。温度均为 Intel 字节序、有符号16位、0.1℃/LSB，字段顺序为 RL、RR、FL、FR：

| CAN ID | 内容 | 控制用途 |
| --- | --- | --- |
| `0x506` | 四个电机温度 | PWM1 使用四个值中的最高值 |
| `0x507` | 四个逆变器温度 | PWM2 控制输入之一 |
| `0x508` | 四个 IGBT 温度 | PWM2 控制输入之一 |

温度有效范围为-40.0℃至200.0℃。默认温控曲线为：

- 温度上升到40℃时开始运行，目标占空比30%；
- 40℃至60℃之间线性增加到100%；
- 60℃及以上保持100%；
- 已经运行后，温度降到35℃以下才停止，避免反复启停。

只要 `0x507` 或 `0x508` 中仍有一个报文有效，PWM2 就继续按有效报文中的最高温度自动控制，同时在诊断报文中标记另一个报文失联。

## 5. 温度报文失联处理

温度报文超过2秒未更新后标记失联。默认处理不再直接切到全速：

1. 已经收到过有效温度时，先保持最后一次自动目标5秒；
2. 保持时间结束后进入“固定保底”策略；
3. 默认保底占空比为 PWM1=50%、PWM2=50%；
4. 如果失联前目标高于50%，继续保持较高目标，不因失联降低原有冷却能力；
5. 上电后从未收到温度时先等待3秒，然后缓慢升到50%保底占空比。

可选失联策略：

| 策略值 | 策略 | 行为 |
| ---: | --- | --- |
| 0 | 保持最后目标 | 有历史目标时一直保持；没有历史目标时使用配置的保底占空比 |
| 1 | 固定保底（默认） | 保持期结束后取“最后目标”和“保底占空比”中的较高值 |
| 2 | 全速 | 目标改为100%，但仍按缓慢启动速度上升 |

失联策略不提供永久关闭选项。需要关闭时使用带有效时间的模式2，避免控制端失联后风扇长期停止。

## 6. CAN 遥控命令

### 6.1 通用格式

命令使用标准帧 `0x5A4`，DLC必须为8：

| 字节 | 内容 |
| --- | --- |
| 0 | 操作码 |
| 1 | 序号，由发送方递增，用于匹配应答 |
| 2..6 | 操作参数 |
| 7 | CRC-8/SAE-J1850，计算范围为字节0..6 |

CRC参数：多项式 `0x1D`、初值 `0xFF`、结果异或 `0xFF`、不反转输入输出。

序号只用于把命令和应答对应起来，不承担防重放或身份认证。CANB应视为车内受控网络。可用下面的代码为任意命令生成CRC：

```python
def crc8_sae_j1850(data):
    crc = 0xFF
    for value in data:
        crc ^= value
        for _ in range(8):
            crc = ((crc << 1) ^ 0x1D) & 0xFF if crc & 0x80 else (crc << 1) & 0xFF
    return crc ^ 0xFF

payload = [0x01, 0x02, 0x01, 40, 50, 10, 0]
frame = payload + [crc8_sae_j1850(payload)]
print(" ".join(f"{value:02X}" for value in frame))
```

控制端应发送一条命令后等待相同序号的 `0x5A5` 应答，再发送下一条策略命令。手动模式的周期续发间隔应明显短于设置的有效时间。

### 6.2 操作码 0x01：设置模式和目标占空比

| 字节 | 内容 |
| --- | --- |
| 2 | 模式：0自动，1手动，2关闭 |
| 3 | PWM1目标占空比，0..100% |
| 4 | PWM2目标占空比，0..100% |
| 5 | 手动/关闭模式有效时间，1..60秒；自动模式忽略 |
| 6 | 保留，必须为0 |

示例：

```text
01 01 00 00 00 00 00 11    回到自动模式
01 02 01 28 32 0A 00 8E    手动 PWM1=40%、PWM2=50%，有效10秒
01 03 02 00 00 05 00 28    两路关闭，有效5秒，然后自动恢复
```

### 6.3 操作码 0x02：设置自动温控曲线

| 字节 | 内容 |
| --- | --- |
| 2 | 关闭温度，整数℃ |
| 3 | 启动温度，整数℃ |
| 4 | 全速温度，整数℃，最大150℃ |
| 5 | 自动模式最低运行占空比，10..100% |
| 6 | 占空比上升速度，10..100%/s |

必须满足“关闭温度 < 启动温度 < 全速温度”。示例为默认35℃/40℃/60℃、最低30%、上升20%/s：

```text
02 04 23 28 3C 1E 14 DC
```

### 6.4 操作码 0x03：设置失联策略

| 字节 | 内容 |
| --- | --- |
| 2 | 失联策略：0保持最后目标，1固定保底，2全速 |
| 3 | PWM1保底占空比，0..100% |
| 4 | PWM2保底占空比，0..100% |
| 5 | 标记失联后继续保持最后目标的时间，0..30秒 |
| 6 | 占空比下降速度，10..100%/s |

默认策略命令为固定保底50%/50%、保持5秒、下降50%/s：

```text
03 05 01 32 32 05 32 B6
```

### 6.5 操作码 0x04/0x05：恢复默认和查询

```text
04 06 A5 00 00 00 00 D7    恢复默认策略并回到自动模式
05 07 00 00 00 00 00 F1    查询当前模式和策略
```

恢复默认命令的字节2必须为 `0xA5`。查询命令返回 `0x5A6` 和 `0x5A7`；温控曲线或失联策略设置成功后返回对应的一帧状态。

## 7. CAN 状态与应答

FanController 使用 `0x5A2` 至 `0x5A7`。主循环收到命令后立即安排应答和策略状态，不等待500ms周期；发送邮箱占满时会在后续主循环重试。

### 7.1 0x5A2 风扇实际状态，500ms周期

| 字节 | 字段 | 类型 | 单位 |
| --- | --- | --- | --- |
| 0..1 | 风扇1实际转速 | `uint16_t`，大端序 | RPM |
| 2..3 | 风扇2实际转速 | `uint16_t`，大端序 | RPM |
| 4..5 | 风扇3实际转速 | `uint16_t`，大端序 | RPM |
| 6 | PWM1当前实际输出占空比 | `uint8_t` | % |
| 7 | PWM2当前实际输出占空比 | `uint8_t` | % |

### 7.2 0x5A3 风扇诊断，500ms周期

| 字节 | 字段 | 说明 |
| --- | --- | --- |
| 0 | 故障位 | bit0..2：风扇1..3无转速；bit3：电机温度超时；bit4：任一控制器温度超时；bit5..6：PWM1/2启动失败；bit7：任一TACH捕获启动失败 |
| 1 | 状态位 | bit0..2：0x506/0x507/0x508有效；bit3：PWM1正在输出；bit4：PWM2正在输出；bit5..6：当前模式 |
| 2..3 | 最高电机温度 | `int16_t`，大端序，0.1℃/LSB；无效时 `0x7FFF` |
| 4..5 | 最高有效控制器温度 | `int16_t`，大端序，0.1℃/LSB；无效时 `0x7FFF` |
| 6 | PWM1目标占空比 | 0..100% |
| 7 | PWM2目标占空比 | 0..100% |

`0x5A2` 的当前输出与 `0x5A3` 的目标值可以直接观察缓慢启动过程。

### 7.3 0x5A5 命令应答

| 字节 | 字段 | 说明 |
| --- | --- | --- |
| 0 | 操作码 | 回显收到的操作码 |
| 1 | 序号 | 回显命令序号 |
| 2 | 结果 | 0成功；1 CRC错误；2长度错误；3参数错误；4不支持的操作码；5手动/关闭模式超时并已回到自动 |
| 3 | 模式和策略 | bit0..1当前模式；bit4..5当前失联策略 |
| 4..5 | PWM1/PWM2当前输出 | 0..100% |
| 6..7 | PWM1/PWM2当前目标 | 0..100% |

### 7.4 0x5A6 自动曲线状态

| 字节 | 内容 |
| --- | --- |
| 0..2 | 关闭、启动、全速温度，整数℃ |
| 3 | 最低运行占空比 |
| 4 | 占空比上升速度，%/s |
| 5..7 | 保留，当前为0 |

### 7.5 0x5A7 失联策略状态

| 字节 | 内容 |
| --- | --- |
| 0 | 失联策略 |
| 1..2 | PWM1、PWM2保底占空比 |
| 3 | 保持最后目标时间，秒 |
| 4 | 占空比下降速度，%/s |
| 5 | 当前模式 |
| 6 | 手动/关闭模式剩余有效时间，秒 |
| 7 | 保留，当前为0 |

## 8. TACH 故障判断

当当前输出达到最低运行占空比后开始计时，连续5秒没有有效TACH转速时设置对应故障位。PWM1同时驱动两个风扇，因此任一个TACH故障都不会关闭整组风扇。

## 9. 台架检查

1. 不接CANB，复位后确认两路保持停止约3秒；PWM1随后以约20%/s升到50%，PWM2在PWM1开始2.5秒后再缓慢启动。
2. 发送 `0x506/0x507/0x508` 的35℃、40℃、50℃、60℃，确认目标依次为0%、30%、约65%、100%，当前输出逐步跟随目标。
3. 只停发 `0x507` 或 `0x508`，确认PWM2继续使用另一个有效温度，但诊断位提示部分失联。
4. 全部停发温度，确认先保持最后目标5秒，再缓慢进入50%保底，而不是直接全速。
5. 发送手动40%/50%命令并周期续发，确认转速可调；停止续发后确认有效时间结束并自动回到温控模式。
6. 发送错误CRC、错误温度顺序和超过100%的占空比，确认 `0x5A5` 分别返回错误且当前策略不变。
7. 分别断开三个TACH，确认运行5秒后设置对应故障位。
